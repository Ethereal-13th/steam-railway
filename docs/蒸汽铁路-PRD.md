## 第1章：产品概述

### 1.1 模组信息

| 项目 | 内容 |
|------|------|
| **模组名称** | 蒸汽铁路（Steam Railway） |
| **模组 ID** | `steam_railway` |
| **目标平台** | Minecraft Java Edition |
| **加载器** | Fabric（第一阶段），架构预留 NeoForge 迁移空间 |
| **目标版本** | 1.21.11+ |
| **授权协议** | MIT |
| **目标场景** | 主打小型朋友服务器（2-10人） |

### 1.2 一句话描述

> 用方块搭建你自己的火车，铺设属于你自己的铁路——然后坐在车里，看着窗外的风景一路向前。即使你离开，它也会在世界的某个角落继续运行。

### 1.3 情绪核心

这个模组不是为了"更快的矿车"。它的情绪核心是**拥有感与持续感**：

- **拥有感**：火车是我自己一砖一瓦搭出来的，线路是我自己铺的。
- **持续感**：这个世界不是围绕我加载的，我的火车在我离开时仍在运行，就像一条真正活着的铁路。

### 1.4 目标玩家画像

| 维度 | 描述 |
|------|------|
| **核心玩家** | 喜欢机械动力（Create）式建造体验、但希望铁路系统更独立更完整的玩家 |
| **建造倾向** | 享受"先搭出好看的东西，再看它动起来"的过程 |
| **游戏风格** | 偏沉浸向、喜欢长线项目、对"世界在我不在时仍在运转"有情感共鸣 |
| **技术接受度** | 能接受比原版矿车更复杂的操作，但不喜欢过度复杂的微管理 |
| **典型场景** | 在自己建的车站月台上等车，坐着火车按规划线路看沿途风景 |

> **关键词**：站点等车、坐火车的仪式感、看沿途风景、线路规划的乐趣

### 1.5 差异化定位

**与机械动力的关系**：

> 以下设计直接受机械动力（Create Mod）启发：
> - 列车底盘沿专用轨道行驶的核心概念
> - 底盘上可粘任意方块搭建外观的建造自由
> - 时刻表驱动自动驾驶的运营模式
>
> 蒸汽铁路的差异化在于：**将这些概念从应力系统中解放出来，为单纯喜欢铁路系统的玩家提供独立选择**。不依赖机械动力的应力网络，只要会铺轨道、搭方块就能跑。

**一句话总结**：
> 机械动力让你造一列火车去**干活**。蒸汽铁路让你造一列火车去**旅行**。

### 1.6 设计铁律

> **铁律一：零原版侵入**
> 
> 蒸汽铁路**不得修改、覆盖或影响任何原版 Minecraft 特性**。原版矿车、原版铁轨、原版实体行为保持100%不变。本模组的所有功能通过新增独立方块、实体和系统实现，与原版系统完全隔离。

> **铁律二：自由上下车，无专用交互点**
> 
> 玩家上下火车的方式是**物理性的**：火车停着或慢速时，玩家直接从车门走进车厢。不需要右键"登车"交互，不需要专用按钮，不需要驾驶位座位。火车是一个整体移动的平台，玩家站在上面就会随火车一起移动，可以在车厢内自由走动。
>
> **实现边界**：自由走动参考机械动力 Contraption 的碰撞处理思路实现：火车实体保存方块结构和简化碰撞体，服务端按火车运动向量推动站在表面的实体，客户端做平滑表现。V0.1 优先保证低速、平缓轨道、少量乘客场景稳定；高速、急弯、多人拥挤、复杂旋转下的边缘问题放入 V0.5 打磨。

## 第2章：核心体验循环

### 2.1 完整体验闭环

```
[获得轨道方块] → [铺设轨道，连接多站点] → [放置站点标记]
      ↓
[放置列车基座] → [在基座上搭建火车外观] → [绑定时刻表/司机方块]
      ↓
[选择模式：自动驾驶 / 手动驾驶]
      ↓
[火车沿轨道运行，途经多个站点] ←──────────┐
      ↓                                    │
[到达站点自动停车 / 手动停靠任意位置]        │
[玩家随时上下车，可在车厢内自由走动]─────────┘（循环）
```

### 2.2 两种驾驶模式

| 模式 | 触发方式 | 体验 |
|------|----------|------|
| **自动驾驶** | 给司机方块放入时刻表 | 火车按站点顺序自动行驶、停靠。玩家可作为乘客在车厢内自由走动 |
| **手动驾驶** | 右键列车基座 | 玩家控制加速/刹车，有惯性的真实手感 |

> **设计原则**：两种模式随时切换。自动驾驶时玩家也可以临时接管手动驾驶；手动驾驶时也可以随时切回自动驾驶。

### 2.3 关键情感峰值（Wow Moments）

| 时刻 | 情感 |
|------|------|
| **第一次把基座放轨道上，它"咔"一声吸附上去** | "这东西能跑！" |
| **在基座上粘完最后一方块，整列火车组装完成** | "这是我造的车" |
| **给司机方块装上时刻表，火车自己启动开走** | "它活了！" |
| **坐在坐垫上穿过自己建的桥梁和隧道** | "这风景是我铺的线路带来的" |
| **离开基地去挖矿，几小时后回来，发现火车正好驶回车站** | "世界真的在运转" |

### 2.4 核心交互节奏

1. **建造阶段**（数分钟~数小时）：铺轨道、搭车站、造火车外观 → **创造力释放**
2. **调试阶段**（数分钟）：设置时刻表、测试运行 → **解决问题的小挑战**
3. **运行阶段**（持续进行）：火车自动运行，玩家随时乘坐 → **沉浸与陪伴感**
4. **扩展阶段**（循环回到1）：修新线路、造新火车、连接新站点 → **长线成长**

### 2.5 火车实体的物理表现

> 火车由"列车基座"作为核心，玩家用**强力胶（Super Glue）** 将任意方块粘贴到基座上，形成一个**整体实体**。
>
> - 视觉上：所有粘贴的方块作为一个整体移动、旋转
> - 物理上：各方块保持各自的碰撞箱（玩家可以在粘贴的方块上走动、跳跃）
> - 行为上：整个组合体沿轨道行驶，像一个真正的移动平台
>
> **实现参考**：采用类似机械动力 Contraption 的结构：静止时是世界方块，激活后转换为一个携带方块结构、座位映射、碰撞体和本地坐标变换的火车实体；解散时再把结构写回世界。

## 第3章：功能模块（分阶段）

### Phase 0: Ghost Train Proof（技术原型）

**目标**：验证核心技术承诺——火车能在未加载区域继续运行，保存重载后状态不丢。

**设计约束**：使用占位方块（无正式材质，可用原版方块或纯色方块代替），无HUD、无音效、无强力胶视觉效果，用调试命令或极简交互完成功能验证，只支持两个站点之间的单一路径。

#### 0.1 轨道系统（原型版）

| 功能 | 说明 |
|------|------|
| **轨道锚点方块** | 放置两个占位方块作为站点A和站点B |
| **路径生成** | 用命令生成两点之间的曲线路径（如 `/sr path create A B`），不铺实际轨道方块 |
| **曲线路径** | 起点和终点不在一条直线上时，自动生成圆滑弯道，不是斜直线 |
| **弯道类型** | 按住 **Ctrl** 生成缓弯（大半径曲线），不按生成急弯（小半径曲线） |
| **路径数据 `RailNetwork`** | 服务端持久化：路径ID、采样节点（位置/切线方向/累计距离）、端点 |
| **调试可视化** | 可选显示路径节点和切线方向供开发验证 |

#### 0.2 列车基座与实体

| 功能 | 说明 |
|------|------|
| **列车基座方块** | 放置在世界中的占位方块 |
| **激活为实体** | 用命令将基座转换为火车实体（如 `/sr train spawn`） |
| **火车实体 `TrainEntity`** | 简化实体（可以只是一个箱子大小的占位模型），沿路径移动，方向随路径切线平滑旋转 |
| **状态绑定** | 实体与路径ID绑定，记录路径进度（0.0 ~ 1.0） |
| **右键驾驶** | 右键列车基座进入手动驾驶模式 |

#### 0.3 时刻表系统（原型版）

| 功能 | 说明 |
|------|------|
| **时刻表物品** | 占位物品，用极简GUI设置起点站→终点站 |
| **GUI设计** | 两个下拉框/输入框（选择A站和B站），一个"应用"按钮 |
| **应用方式** | 将时刻表绑定到火车实体 |
| **自动行驶** | 火车沿路径从当前位置向目标站点移动，到达后停止 |

#### 0.4 手动驾驶（原型版）

| 功能 | 说明 |
|------|------|
| **控制方式** | 右键列车基座后，W/S 控制加速/刹车 |
| **惯性手感** | 加速/减速有渐变过程，验证物理手感 |

#### 0.5 后台模拟系统（核心技术）

| 功能 | 说明 |
|------|------|
| **逻辑状态 `TrainState`** | 服务端持久化：列车ID、路径ID、路径进度、速度、目标站点、运行模式 |
| **存档持久化** | 世界保存时将 `TrainState` 写入独立世界存档数据（`steam_railway` 命名空间），重载后恢复 |
| **未加载区域模拟（精确模拟）** | 区块卸载后，由全局铁路管理器在服务端 tick 中继续推进火车逻辑位置（纯数学推算：位置 = 原位置 + 速度 × 时间）。不加载区块、不做方块碰撞、不与世界交互 |
| **逻辑与世界操作隔离** | 列车运行状态不依赖区块内实体持续存在；世界方块操作、实体生成/移动、网络同步仍在服务端主线程执行。Phase 0 不引入独立计算线程，后续只有在性能测试证明必要时再评估后台 worker |
| **状态同步** | 区块重新加载时，根据逻辑状态生成/更新可见实体，玩家看到火车在正确位置 |

#### Phase 0 验收标准

| # | 验收项 | 验证方式 |
|---|--------|----------|
| 1 | 放置两个轨道锚点，不在一条直线上，生成圆滑曲线路径 | 调试可视化确认 |
| 2 | 按住Ctrl生成缓弯，不按生成急弯 | 对比两种路径的曲率半径 |
| 3 | 列车基座转换为实体，沿曲线路径移动，方向平滑旋转 | 目视观察 |
| 4 | 极简GUI设置A→B时刻表，火车自动驾驶到B站停止 | 功能测试 |
| 5 | 保存→退出→重载，火车状态和位置不丢失 | 存档验证 |
| 6 | 玩家远离使区块卸载，返回后火车位置已按逻辑推进 | 距离对比测试 |
| 7 | 手动控制加速/减速有惯性手感 | 操作感受确认 |

### Phase 1: V0.1 最小可玩版本

**目标**：玩家能用方块搭出一列火车，在铺设的轨道上自动/手动运行，保存重载后状态不丢，能在车厢内走动或坐在坐垫上看风景。

**设计约束**：基础材质、无HUD、无音效、无燃料系统。

#### 1.1 手持铺轨工具

| 功能 | 说明 |
|------|------|
| **工具物品** | `轨道铺设工具`（Rail Placement Tool） |
| **轨道类型选择** | 工具自带轨道类型选择模式（如按快捷键切换，或工具GUI选择）。Phase 1 只支持标准轨道，后续版本加入窄轨/宽轨等 |
| **消耗轨道物品** | 铺设轨道时，从玩家背包中消耗对应类型的轨道物品。预览线显示需要消耗的数量，确认铺设时扣除 |
| **辅助放置** | 手持工具，右键已放置的轨道或地面设定起点 |
| **实时预览** | 选定起点后，玩家准星对着的方块实时渲染为终点，预览轨道走线（两根铁轨的线条）。预览线随时跟随准星移动 |
| **曲线预览** | 按住 **Ctrl** 时，预览线实时显示缓弯效果；不按显示急弯效果 |
| **确认铺设** | 对准目标位置再次右键，沿预览路径生成实际轨道方块，同时从背包扣除轨道物品 |
| **取消预览** | **Shift+右键**取消当前起点，或切换工具取消预览 |
| **副手填充** | 副手持方块时，自动用该方块填充轨道下方的空气；无副手方块时，轨道贴合地面/现有方块 |
| **对齐轨道** | 8个标准方向的轨道可单独放置/破坏；曲线轨道为非对齐轨道，只能作为集合整体操作 |
| **轨道图注册** | 确认铺设后生成可持久化的 track segment，并注册到 `RailNetwork` 的节点/边图中。运行时列车沿轨道图采样点移动，不每 tick 从世界方块重新推断轨道 |

> **消耗逻辑**：预览时计算需要多少格轨道，如果背包数量不足，预览线显示为红色（不可铺设）；数量充足显示为绿色。确认铺设后从背包扣除。

#### 1.2 标准轨道方块

| 功能 | 说明 |
|------|------|
| **手动放置** | 右键地面放置轨道，朝向远离玩家 |
| **放置规则** | 可放在固体方块顶部，或形成斜坡 |
| **坡度规则** | 不要太陡。水平 **5格** 距离下降 **1格**（约11.3度坡度）。数值可后续调整 |
| **自动连接** | 相邻轨道自动连接形成连续路径，视觉上平滑过渡 |
| **路径注册** | 放置时向 `RailNetwork` 注册，拆除时注销 |
| **视觉效果** | 参考机械动力的轨道渲染：平滑曲线、斜坡过渡自然 |

#### 1.3 站点方块

| 功能 | 说明 |
|------|------|
| **方块放置** | 放置在轨道旁边（或轨道上方），标记站点位置 |
| **站点命名** | 右键打开输入框，命名站点（如"中央车站"、"矿场站"） |
| **自动停靠** | 火车运行到站点位置时自动减速停车 |
| **停靠时间** | 时刻表可设置每个站点的停靠时间（秒） |

#### 1.4 强力胶系统（Super Glue）

| 功能 | 说明 |
|------|------|
| **Block by Block 模式** | 手持强力胶右键方块的一个面，给该面"上胶"，然后可在该面粘贴其他方块 |
| **Area 模式** | 右键选一个角，移动至对角（绿色框预览范围），再右键完成粘合。范围内所有方块形成一个连通组 |
| **连通组扩展（传递性）** | 已属于连通组的任何方块，都可以继续作为新的粘合起点。给连通组内某个方块的上胶面粘新方块 → 新方块自动加入该连通组 → 新方块也可以继续粘其他方块。组合体可以无限扩展 |
| **副手自动粘合** | 强力胶放副手，放置方块时自动粘合到相邻的已上胶面 |
| **碰撞保持** | 各方块保持原碰撞箱，玩家可以在粘合的方块上走动、跳跃 |
| **绿色轮廓** | 手持强力胶时，已粘合的连通组显示绿色轮廓 |
| **解除粘合** | 手持强力胶，左键（攻击键）点击已粘合区域即可解除该方块的粘合关系 |
| **不做自动扫描** | 火车实体化时只收集玩家显式粘合到列车基座连通组内的方块，不自动扫描周围方块。避免玩家旁边的建筑、站台、装饰被误收入火车 |
| **实现参考** | 参考机械动力 Super Glue：客户端显示选择框和可达连通组，确认后向服务端发送选择结果，服务端生成胶水连接实体/记录；攻击已选胶水时移除连接 |

#### 1.5 列车基座（转向架）

| 功能 | 说明 |
|------|------|
| **手持外观** | 像普通方块一样 |
| **放置效果** | 放置在轨道上时，显示为有轮子的底盘外观（参考机械动力的Bogey转向架） |
| **静止状态** | 是一个普通方块，周围可以粘贴其他方块，玩家可以在上面建造火车 |
| **激活为实体** | 玩家右键列车基座，基座+所有强力胶连通方块一起转换为火车实体 |
| **运行状态** | 火车作为一个整体实体沿轨道移动，各方块随基座一起运动 |
| **解散还原** | 火车停稳后，可解散为静态方块组，玩家可以继续修改 |
| **方向保持** | 实体化时，火车方向与轨道切线方向对齐 |

#### 1.6 司机方块（Train Driver）

| 功能 | 说明 |
|------|------|

## 第4章：技术架构

### 4.1 命名空间与数据隔离

| 项目 | 说明 |
|------|------|
| **命名空间** | `steam_railway` |
| **数据隔离原则** | 所有铁路状态写入独立的世界存档数据，不写入原版矿车、铁轨或无关方块 NBT |
| **铁律** | **零原版侵入**：不修改、覆盖或影响任何原版 Minecraft 特性 |

### 4.2 核心数据模型

| 数据类型 | 存储内容 |
|----------|----------|
| **`RailNetwork`** | 路径ID、采样节点（位置/切线方向/累计距离/限速）、端点、道岔引用 |
| **`TrainState`** | 列车ID、当前路径ID、路径进度、速度、时刻表索引、运行模式（自动/手动）、司机方块状态 |
| **`ScheduleState`** | 站点位置、站点间绑定的路径ID、停靠时间、道岔选择 |
| **`ContraptionState`** | 火车方块组的结构信息（方块类型、相对位置、显式强力胶连接关系、座位映射、简化碰撞体） |

### 4.3 两层运行模型

```
┌─────────────────────────────────────────────────────────┐
│                      服务端主线程                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐  │
│  │ 世界方块操作 │    │ 实体生成/移动 │    │  网络同步   │  │
│  └─────────────┘    └─────────────┘    └─────────────┘  │
└─────────────────────────────────────────────────────────┘
                            ↑
                    状态同步（区块加载时）
                            ↓
┌─────────────────────────────────────────────────────────┐
│                 全局铁路管理器（服务端 tick）                 │
│  ┌─────────────────────────────────────────────────┐    │
│  │              纯数学推算（无世界交互）               │    │
│  │  位置 = 原位置 + 速度 × 时间                      │    │
│  │  速度 = f(加速度, 轨道限速, 下坡/上坡)             │    │
│  │  方向 = 路径切线方向                              │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

| 区域状态 | 表现 |
|----------|------|
| **已加载区域** | 存在可见的 `TrainEntity`，负责渲染、交互、正常移动 |
| **未加载区域** | 服务端只保存轻量级 `TrainState`，全局铁路管理器做纯状态推进 |
| **重新加载** | 根据 `TrainState` 生成/更新可见实体，玩家看到正确位置 |

> **Create 式运行原则**：列车逻辑由全局管理器统一推进，状态不依赖区块实体是否加载。Phase 0 先在服务端 tick 内完成纯状态计算，避免过早引入多线程复杂度；所有世界操作（实体生成/移动、方块修改、网络同步）保持在服务端主线程执行。

### 4.4 路径表示

第一版把铁路保存为**轨道图 + 采样路径节点**。参考机械动力的 TrackGraph 思路：轨道网络由节点、边、边上附加点组成；列车运行时沿图上的边和采样点推进，而不是每 tick 从世界轨道方块重新推断。

| 节点属性 | 说明 |
|----------|------|
| **位置** | 世界坐标 (x, y, z) |
| **切线方向** | 火车在该点的朝向 |
| **累计距离** | 从起点开始的轨道长度 |
| **限速** | 该段轨道的最高速度 |
| **道岔引用** | 后续可选的道岔分支 |

> 这样列车移动、到站停车、未加载区域模拟都不需要每 tick 从世界方块里重新推断整条轨道。
>
> **Phase 1 限制**：轨道图可以先支持连续路径和简单环线；节点/边数据结构预留道岔引用，但分支选择逻辑到 Phase 3 实现。

### 4.5 网络同步策略

| 同步内容 | 方式 |
|----------|------|
| **火车位置** | 服务端权威。附近客户端收到周期性位置/速度/路径进度快照并插值显示；启动、停车、换路径、重新加载等关键事件立即同步 |
| **乘客位置** | 坐垫乘客使用原版 riding/passenger 同步；自由站立玩家由服务端碰撞修正位置，客户端做平滑表现 |
| **轨道更新** | 轨道放置/拆除时同步轨道图增量；客户端只用作渲染和预览，服务端轨道图是权威 |
| **驾驶输入** | 客户端发送 W/S 按键状态，服务端计算速度 |

> **同步原则**：不每 tick 广播完整 `TrainState`。高频状态用轻量快照和客户端插值，低频结构变更用可靠增量包。多人测试后再调整同步频率。

### 4.6 平台策略

| 项目 | 说明 |
|------|------|
| **第一阶段** | Fabric 独占开发 |
| **未来兼容** | 数据模型和逻辑层保持与平台 API 解耦，预留 NeoForge 迁移空间 |

## 第6章：美术与表现

### 6.1 视觉风格方向

| 维度 | 方向 |
|------|------|
| **整体风格** | 工业革命/蒸汽朋克，偏向机械动力的工业美学 |
| **色彩基调** | 金属灰、木棕、锈红为主，点缀黄铜色 |
| **细节程度** | 中高精度——比原版细腻，但不追求照片级真实 |

### 6.2 材质分辨率与规范

| 项目 | 规范 |
|------|------|
| **基础分辨率** | 16×16（与原版一致），关键方块可 32×32 |
| **风格一致性** | 所有材质保持统一的像素风格和明暗处理 |
| **透明度使用** | 轮子、支架等可使用透明度实现镂空效果 |

### 6.3 音效设计原则

| 场景 | 音效风格 |
|------|----------|
| **行驶声** | 低频"哐当"节奏感，速度越快频率越高 |
| **机械声** | 中频齿轮/蒸汽声，增加工业感 |
| **环境声** | 鸣笛、风声、站点铃声 |
| **交互声** | 铺轨确认音、粘合音、组装/解散音 |

### 6.4 动画需求

| 动画 | 触发条件 | 首次出现 |
|------|----------|----------|
| **轮子转动** | 火车运行时，速度与转动速度成正比 | Phase 3 |
| **车体晃动** | 火车运行时持续轻微晃动 | Phase 3 |
| **道岔切换** | 道岔方向改变时 | Phase 3 |
| **启动烟尘** | 火车从静止启动时 | Phase 2 |
| **刹车火花** | 急刹车时 | Phase 2 |

## 第7章：范围控制（明确不做）

以下功能在 V1.0 及之前版本**明确不做**：

| 功能 | 不做原因 | 未来可能性 |
|------|----------|-----------|
| **信号灯系统** | 复杂度高，V1.0 聚焦核心体验 | V1.x 可能加入 |
| **多节车厢挂钩** | 涉及复杂的物理连接和碰撞 | V1.x 可能加入 |
| **列车碰撞** | 需要精确的实体碰撞检测 | V1.x 可能加入 |
| **红石接口（完整）** | V1.0 只做道岔红石控制，不做完整红石联动 | V1.x 可能扩展 |
| **机械动力应力供能联动** | 本模组是独立系统，不依赖其他模组的能量系统 | 不会做 |
| **货运/容器系统** | V1.0 聚焦客运体验 | V1.x 可能加入 |
| **自然生成车站** | 与建造核心冲突 | 不会做 |
| **轨道磨损效果** | 视觉收益低，实现复杂 | 不会做 |
| **燃料系统** | 简化 V0.1 复杂度，后续再评估 | V0.5+ 可能加入 |

## 第9章：风险与待决策事项

### 9.1 已确认决策

| 问题 | 决策 |
|------|------|
| 未加载火车模拟方式 | **精确模拟**，全局铁路管理器在服务端 tick 中推进状态 |
| 平台选择 | **Fabric 优先**，架构预留 NeoForge 迁移空间 |
| 燃料系统 | **V1.0 之前不做** |
| 速度决定因素 | **轨道类型**决定 |
| 驾驶与乘坐分离 | 驾驶时固定位置，乘坐用坐垫系统 |
| 目标场景 | **主打小型朋友服务器**（2-10人） |
| 致敬边界 | **为单纯喜欢铁路系统的玩家提供独立选择** |
| 火车最大长度 | **Phase 1 默认 64 方块**；128 方块作为配置上限或压测后开放目标 |
| 单次铺设最大距离 | **50 格** |
| 开源协议 | **MIT** |
| 强力胶收集方式 | **玩家显式粘合**，不做自动扫描吸附 |
| Phase 1 路线范围 | **连续线路/简单环线**，分支和道岔路线选择放到 Phase 3 |
| 跨加载器策略 | **先保证 Fabric 第一版跑通**，逻辑层尽量解耦平台 API，但不为 NeoForge 预留牺牲首版速度 |

### 9.2 待确认问题

| 问题 | 影响 | 建议 |
|------|------|------|
| **联机同步优化** | 小型朋友服务器中延迟和同步体验 | 需要实际测试多人并发 |
| **合成配方平衡** | 轨道/工具/方块的成本是否合理 | 需要实际游玩测试后调整 |
