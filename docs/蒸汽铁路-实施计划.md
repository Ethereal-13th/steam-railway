## 1. 当前实现边界

### 1.1 轨道系统

- 真实直线轨道：使用 `StandardRailBlock` 连续铺设
- 合法弯道：采用 Create 风格的“直线优先 + 最小连续曲线”
- 曲线中段：当前由曲线连接渲染 + 隐形支撑块承载
- 客户端预览、服务端真实铺设、`RailNetwork` 写入都统一依赖 `RailPathGenerator`

### 1.2 列车系统

- `TrainState` 是列车权威状态
- `TrainEntity` 是可见表现层
- 逻辑推进不依赖实体持续存在
- 玩家离开区块后，列车逻辑继续推进；返回后按状态恢复

### 1.3 本地化规则

- 所有玩家可见文本必须使用翻译 key
- 语言资源至少同时维护 `zh_cn` 与 `en_us`
- 不接受硬编码英文提示

---

## 2. P1a 收口定义

当前仓库将以下条件视为 `P1a` 完成门槛：

- 直线轨道四个方向都可真实铺设
- 合法弯道可真实铺设
- 非法弯道稳定显示红色无效预览，并在确认时拒绝铺设
- 客户端预览与最终铺设结果一致
- 直线段优先保留为真实方块，不整段误曲线化
- 覆盖/拆除后不会残留旧 `CurveRailConnection`、支撑块或失效 `RailNetwork edge`
- `zh_cn` 与 `en_us` 下的名称、提示、命令反馈都可读
- `./gradlew build --console=plain` 通过

### 当前仍保留的实现边界

- 弯道中段不是完整实体 block 系统
- 当前优先级是“轨道与网络一致性”，不是最终美术终稿

---

## 3. 系统边界

### 3.1 模块职责

- `rail/`
  - 轨道方块
  - 路径求解
  - 铺轨工具
  - 轨道网络
- `train/`
  - 列车状态
  - 列车推进
  - 列车可见实体
- `contraption/`
  - 基座
  - 显式强力胶
  - 列车结构组装/解散
- `persistence/`
  - 世界级持久化
- `client/`
  - 客户端预览与曲线渲染

### 3.2 禁止混层

- 不在客户端渲染代码里维护权威状态
- 不在 `rail/` 中直接写乘客或驾驶逻辑
- 不在 `train/` 中直接解析胶水结构

---

## 4. 下一阶段：P1b

### 4.1 目标

把“轨道可铺、列车可跑”的原型推进到“玩家可以自己把方块组装成列车”。

### 4.2 交付物

- 列车基座
- 显式强力胶
- 结构遍历与结构快照
- 激活后的列车结构实体化
- 解散后的世界恢复

### 4.3 范围边界

`P1b` 只解决：

- 哪些方块属于一列车
- 玩家如何显式组装
- 玩家如何安全解散并写回世界

`P1b` 不解决：

- 手动驾驶
- 时刻表
- 乘坐与自由走动
- 道岔、多轨距、红石扩展

---

## 5. 推荐执行顺序

后续默认按下面顺序推进：

1. `P1b` 基座
2. `P1b` 显式强力胶
3. `P1b` 列车结构组装/解散
4. `P1c` 手动驾驶与时刻表
5. `P1d` 坐垫乘坐与低速自由走动

如果用户只要求修回归，优先限定在当前阶段，不顺手跨进后续阶段。

---

## 6. 测试策略

### 6.1 当前必须保留的回归点

- 世界重载后路径不丢
- 世界重载后列车状态不丢
- 区块卸载后列车逻辑继续推进
- 玩家返回后列车位置不倒退
- 轨道覆盖/拆除后不残留失效连接或失效网络边
- `zh_cn` 与 `en_us` 下玩家可见文本不出现翻译 key 泄漏

### 6.2 P1b 新增测试重点

- 只收集显式粘合方块
- 不误吸收周边建筑
- 解散后不吞方块、不复制方块
- 存档重载后结构状态一致

## 7. 当前默认工作协议

每次开始工作前：

1. 确认当前阶段
2. 明确本次只处理哪个子模块
3. 检查是否跨阶段；若跨阶段则收缩范围

每次结束工作时：

1. 说明本次完成了什么
2. 说明还剩什么
3. 说明当前阻塞点
4. 给出下一步最合理的任务
